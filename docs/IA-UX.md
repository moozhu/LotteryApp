### 1.2 页面层级

**核心原则：只有三个页面层级**

| 层级 | 页面 | 路径 | 说明 |
|-----|------|------|------|
| 1 | **首页** | `/` | 应用入口，奖项选择 |
| 2 | **抽奖页** | `/draw/:prizeId` | 从首页奖项卡片进入，抽奖动画 |
| 2 | **设置页** | `/settings` | 从首页或快捷栏进入，配置管理 |

**弹窗（非页面）**:
- 中奖名单弹窗 - 从首页快捷栏打开
- 重置确认弹窗 - 从首页快捷栏或设置页打开
- 首次使用引导弹窗 - 检测到无参与者时自动弹出

## 2. 页面流程

### 2.1 核心流程：首次使用与空状态处理

```
┌─────────┐    ┌─────────────┐    ┌─────────────────────┐    ┌─────────────┐
│  打开   │───▶│ 检测无参与  │───▶│  显示引导弹窗       │───▶│  首页       │
│  应用   │    │  者时弹出   │    │  "请先设置参与者"   │    │  (默认奖项) │
└─────────┘    └─────────────┘    └─────────────────────┘    └─────────────┘
                                        │                           │
                                        ▼                           ▼
                              ┌─────────────────────┐      ┌───────────────┐
                              │  点击"前往设置"     │      │ 点击奖项卡片  │
                              │  跳转到设置页       │      │ 检测无参与者  │
                              │  自动打开参与者Tab  │      │ 再次弹出引导  │
                              └─────────────────────┘      └───────────────┘
```

**空状态规则**:
1. **默认状态**: 无论是否有参与者，首页始终显示4个默认奖项（一等奖到四等奖）
2. **引导弹窗**: 
   - 首次打开应用时，如果无参与者，自动弹出引导弹窗
   - 用户关闭弹窗后，仍可看到4个默认奖项卡片
3. **点击拦截**: 
   - 点击奖项卡片时，检测是否有参与者
   - 如果没有参与者，再次弹出引导弹窗
   - 如果有参与者，正常进入抽奖页面
4. **手动返回**: 
   - 用户设置参与者后，需要手动返回首页
   - 原因：用户可能需要对导入的名单进行编辑、删除等操作
   - 用户主动确认名单无误后，再返回首页抽奖

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  首页   │───▶│ 选择奖项 │───▶│ 抽奖页面 │───▶│ 点击抽奖 │───▶│ 动画播放 │
│         │    │         │    │         │    │         │    │         │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └────┬────┘
                                                                  │
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐         │
│ 返回首页 │◀───│ 展示结果 │◀───│ 彩带特效 │◀───│ 中奖音效 │◀────────┘
│         │    │         │    │         │    │         │
└─────────┘    └─────────┘    └─────────┘    └─────────┘
```

### 2.3 核心流程：重置数据

```
┌─────────┐    ┌─────────────────────┐    ┌─────────────────────────────┐
│ 点击    │───▶│ 显示确认弹窗        │───▶│ 用户选择重置选项            │
│ 重置按钮│    │ "确定要重置数据吗？" │    │ ☑ 清除中奖记录（默认勾选）  │
└─────────┘    │                     │    │ ☐ 清除参与者                │
               │ 显示警告信息        │    │ ☐ 清除奖项                  │
               │ 说明重置后果        │    │                             │
               └─────────────────────┘    └──────────────┬──────────────┘
                                                        │
                              ┌─────────────────────────┼─────────────────┐
                              ▼                         ▼                 ▼
                    ┌─────────────────┐      ┌─────────────────┐  ┌─────────────────┐
                    │ 仅清除中奖记录  │      │ 清除中奖+参与者 │  │ 全部清除        │
                    │ 保留参与者和奖项│      │ 保留奖项        │  │ 恢复初始状态    │
                    │ 可继续抽奖      │      │ 需重新设置参与者│  │ 需重新配置所有  │
                    └─────────────────┘      └─────────────────┘  └─────────────────┘
```

**重置选项说明**:
- **清除中奖记录**: 默认勾选，所有中奖记录清空，可重新抽奖
- **清除参与者**: 人员列表清空，需要重新导入或添加
- **清除奖项**: 奖项列表恢复为默认值（一等奖到五等奖）
- **全部清除**: 相当于恢复出厂设置，回到首次使用状态

#### 3.1.2 响应式方案（基于视口单位）

**设计原则**: 主要面向 PC 端大屏展示（投影/电视），使用现代视口单位实现流畅缩放

**推荐方案: 流体布局 + 视口单位**

```css
/* 基础字体大小使用 clamp 实现流体缩放 */
:root {
  /* 字体大小: 在 1366px 到 2560px 之间平滑缩放 */
  --text-base: clamp(14px, 0.8vw + 8px, 18px);
  --text-lg: clamp(16px, 1vw + 8px, 20px);
  --text-xl: clamp(20px, 1.5vw + 8px, 28px);
  --text-2xl: clamp(24px, 2vw + 8px, 36px);
  --text-3xl: clamp(32px, 3vw + 8px, 48px);
  --text-4xl: clamp(40px, 4vw + 8px, 72px);
  
  /* 间距使用 vmin 保持比例 */
  --space-sm: clamp(8px, 1vmin, 12px);
  --space-md: clamp(16px, 2vmin, 24px);
  --space-lg: clamp(24px, 3vmin, 40px);
  --space-xl: clamp(32px, 4vmin, 64px);
  
  /* 组件尺寸 */
  --header-height: clamp(56px, 6vh, 80px);
  --card-min-width: clamp(240px, 18vw, 320px);
  --card-gap: clamp(16px, 2vw, 32px);
}

/* 快捷操作栏 */
.quick-actions {
  height: var(--header-height);
  padding: 0 var(--space-md);
}

/* 主标题 */
.main-title {
  font-size: var(--text-4xl);
  margin-bottom: var(--space-sm);
}

.sub-title {
  font-size: var(--text-lg);
  margin-bottom: var(--space-lg);
}

/* 奖项网格 */
.prize-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(var(--card-min-width), 1fr));
  gap: var(--card-gap);
  padding: var(--space-md);
}
```

**优势**:
- ✅ 无级缩放，任意分辨率都清晰
- ✅ 基于视口比例，大屏展示效果更好
- ✅ 无需断点，代码更简洁
- ✅ 投影/电视场景下自动适配

**最小支持尺寸**: 1280x720 (720p 投影)
**最佳展示尺寸**: 1920x1080 (1080p 电视/投影)
**理想展示尺寸**: 2560x1440 或 3840x2160 (2K/4K 大屏)

**移动端适配（可选）**:
```css
@media (max-width: 768px) {
  :root {
    --text-base: 14px;
    --card-min-width: 280px;
  }
  
  .prize-grid {
    grid-template-columns: 1fr;
  }
}
```

## 5. 动效规范

### 5.1 时间函数

| 名称 | 值 | 使用场景 |
|-----|---|---------|
| ease-smooth | cubic-bezier(0.4, 0, 0.2, 1) | 页面过渡、卡片悬停 |
| ease-bounce | cubic-bezier(0.68, -0.55, 0.265, 1.55) | 弹窗、按钮点击 |
| ease-accelerate | cubic-bezier(0.4, 0, 1, 1) | 抽奖加速 |
| ease-decelerate | cubic-bezier(0, 0, 0.2, 1) | 抽奖减速 |

### 5.2 持续时间

| 类型 | 时长 | 使用场景 |
|-----|------|---------|
| instant | 0ms | 状态切换 |
| fast | 150ms | 按钮反馈、颜色变化 |
| normal | 300ms | 悬停效果、展开/收起 |
| slow | 500ms | 页面过渡、卡片动画 |
| dramatic | 1000ms+ | 抽奖动画、特效展示 |

## 10. 附录

### 10.1 设计原则

1. **仪式感**: 抽奖是特殊时刻，界面需要有庆祝氛围
2. **即时反馈**: 每个操作都需要有视觉或听觉反馈
3. **容错性**: 重要操作需要确认，支持撤销
4. **无障碍**: 考虑色盲用户，不依赖颜色传递信息
5. **性能优先**: 动画流畅，不卡顿
